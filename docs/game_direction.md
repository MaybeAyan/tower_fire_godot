# 苍纹牌阵方向总纲

## 1. 这款游戏是什么

《苍纹牌阵》是一款剧情驱动的角色战棋。它的核心不是复刻《火焰纹章》的高随机攻防，也不是重卡牌构筑，而是通过逐步入队的小队成员、清晰的职业职责、可预判的战斗风险、战间营地整备与章节叙事，让玩家对角色与队伍形成长期投入。

一句话定义：

- 角色成长战棋
- 少量技能装备
- 强剧情推进
- 阵型协同优先
- 正交规则 + 2.5D 表现

## 2. 玩家为什么会留下来

玩家留下来的主要原因不是刷数值，而是以下四件事同时成立：

1. 角色有记忆点  
   每名主要角色都必须兼具战斗身份与剧情身份，不能只做系统单位。

2. 战斗有位置决策  
   站位、相邻、夹击、保护线、危险区与地形读取必须持续影响玩家选择。

3. 成长有长期取舍  
   每名角色主动技能保持少量精简，重点放在被动、职责与营地整备选择上。

4. 章节有推进感  
   玩家能感受到“从守住圣庭，到追击回廊，再到形成真正的阵型协同小队”的成长曲线。

## 3. 与火焰纹章的差异

《苍纹牌阵》不以高随机命中、闪避、暴击为主要风险来源。首版风险来自确定性规则：

- 反击
- 破绽
- 护卫
- 阵型协同
- 高台、治疗格、危险格、瓶颈区

本作强调：

- 可预测的攻防后果
- 小队角色之间的组合关系
- 章节任务目标变化
- 角色关系与战斗联动

本作不强调：

- 高随机命中博弈
- 大量武器克制表
- 早期复杂职业树
- 大量战斗内抽牌循环

## 4. 核心卖点

### 角色驱动

- 阿斯特拉是指挥与收束核心。
- 莉奥拉负责支援与稳定战线。
- 凯尔负责承伤、卡位与护卫。
- 伊芙琳负责侧翼、破绽与节奏变化。
- 赛琳负责把相邻与共鸣变成全队级战术表达。

### 阵型驱动

战斗特色不是“谁单挑更强”，而是“谁站在哪里、与谁形成关系”。首版只保留最能读懂的阵型规则：

- 相邻增益
- 夹击制造破绽
- 指挥/旗手触发共鸣

### 章节驱动

章节不是单纯换地图打下一场，而是让玩家在剧情、招募、营地、成长与部署上持续做选择。

## 5. 成长原则

前三章的成长重心固定为：

- 强被动
- 少量主动技能
- 低复杂度装备位
- 明确职业职责

前三章不追求：

- 大量技能牌池
- 复杂职业派生
- 大量装备与词条
- 全面展开的多系统叠层

## 6. 章节节奏原则

- 第 1 章：教学章，先守住圣庭，开局只给阿斯特拉和莉奥拉。
- 第 2 章：扩队章，引入凯尔的护卫与伊芙琳的侧翼。
- 第 3 章：阵型章，引入赛琳，让协同成为主玩法而不是加分项。

前三章固定为每章 2 战。每章新增 1 名可长期投入的角色。

部署人数随章节递增：

- 第 1 章：2 人
- 第 2 章：3 人
- 第 3 章：4 人

## 7. 视觉与表现原则

- 底层规则继续使用正交格，不改坐标逻辑。
- 表现层向 2.5D 靠拢，突出地形层次、高低差与镜头氛围。
- UI 与棋盘可读性优先，不牺牲战斗信息密度换纯装饰感。
- 每个新系统先出至少 1 张概念图，再进入实现。

## 8. 开发原则

- 先文档，后概念图，再小步实现。
- 不一次性扩很多系统，先把特色打透。
- 所有结构性 UI 修改默认纳入 Hastur 插件工作流。
- 所有实现任务必须能回到 `tools/check_all.ps1` 做基本验证。
