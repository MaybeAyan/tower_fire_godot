# 苍纹牌阵美术与界面规范

## 1. 风格锚点

《苍纹牌阵》的正式方向固定为：

- 明亮奇幻战场
- 高清像素战棋单位
- 日式油画主导的大面板 UI
- 日式幻想战棋的角色成长、牺牲与章节节奏感

战场不是阴沉地牢，而是晨辉庭院、圣庭回廊、礼拜露台这类清亮但有战压的场所。背景允许柔焦、轻微暗角和景深，但不能压暗到吞掉棋盘信息。

UI 不是厚重黑金中世纪框，也不再以蓝灰毛玻璃大框为主，而是偏日式幻想油画的纸本、笔触、暖金边饰和清晰安全文字区。棋盘信息层可保留轻量玻璃语汇。整体关键词：

- 温润
- 有叙事感
- 有层次
- 可读
- 有仪式感但不过度繁复

## 2. 视觉语汇

### UI 材质规则

- 章节、营地、奖励、结算和对话的大面板统一使用日式油画纸本/暖褐底/金边框体。
- 棋盘高亮、移动范围、危险范围等信息层可以继续使用轻量玻璃与冷光语义。
- 金色用于角饰、当前重点、章节完成、胜利确认和少量分隔线，不作为大面积底色。
- 面板内部允许笔触感、纸纹和雾面质感，但文字安全区必须保持低噪声、低装饰。
- 所有正式 UI 资源必须无文字，所有文案由 Godot 运行时绘制。

### 角色表现规则

- 主角组必须一眼区分：阿斯特拉偏锋锐纹章剑士、莉奥拉偏圣辉支援、凯尔偏誓盾护卫。
- 敌方职业也必须一眼区分：剑士偏赤刃突进、法师偏符焰远程、护卫偏厚重盾墙。
- 头像、立绘、棋盘动作表、卡图、技能特效必须共享同一职业语义，不允许“头像像圣职、棋盘像盗贼、卡图像法师”的错位。
- 单位动作表以清晰站姿、强起手、明确受击、利落收招为优先，不追求花哨但模糊的动画。

### 战斗语义色规则

- 我方主强调：苍青 / 冰蓝
- 我方支援与治疗：圣辉薄金 / 薄青绿
- 敌方威胁：赤红 / 洋红
- 守护与格挡：青绿 / 冷青
- 教学与关键目标：淡金

语义色只承担信息优先级，不应整屏泛滥。棋盘高亮必须能看见地形。

## 3. 场景与界面统一规则

### 战斗 HUD

- 上方提示、中央棋盘、右侧状态栏、底部技能栏必须明显分层。
- 左上教学卡和右上目标提示应像“悬浮战术卡”，不能压进主棋盘核心视野。
- 底部技能区使用横向技能条，不回到厚重大卡堆叠。
- 右侧信息必须分成固定区块：回合头、单位区、目标区、战况区、底部行动区。

### 对话、奖励、营地、章节地图

- 虽然功能不同，但必须共用同一套油画纸本大框语言。
- 对话框允许更强的叙事感，但禁止回退到旧中世纪厚框。
- 奖励与转职界面应显得“珍贵但清爽”，重点靠构图和局部金饰，不靠大面积黑金压迫。
- 营地和章节地图的 UI 要像战役管理界面，不像另一款游戏的菜单页。

## 4. 文字与排版规则

- 中文优先保证信息清晰，不挤字，不堆叠，不硬塞一行。
- 所有运行时文本必须先限宽，再缩短或降字号。
- 英雄级大标题只用于章节、胜利、转职这类关键时刻。
- 侧栏、技能栏、提示卡内标题必须收紧，不做过大的展示字。

## 5. 第一章正式界面清单

第一章成品切片必须统一到正式风格的界面：

- 开场对白界面
- 战斗右侧战术面板
- 底部技能栏
- 左上教学卡
- 右上目标提示
- 战斗内短提示 / 反馈卡
- 战后结算
- 奖励选择
- 转职界面
- 章节地图
- 营地详情
- 章节完成界面

## 6. 第一章正式角色清单

第一章成品切片优先做透这些角色的表现：

- 阿斯特拉
- 莉奥拉
- 凯尔
- 伊芙琳
- 赤刃侍从
- 符焰术士
- 誓盾卫士
- 赤刃队长

### 重绘或重做一致化

- 奖励面板正式框体
- 对话框正式框体
- 营地与章节地图专用大面板
- 小按钮体系
- 章节完成 / 转职强调面板
- 角色头像与动作表中辨识度偏弱的条目

## 8. 工程规则

- `art_assets.json` 只保留正式在用映射，不再让旧风格资源与正式资源并行混用。
- 所有新增资源必须保留三层：
  - 运行时资源
  - `source/` 源文件
  - 过程元数据，如 `manifest.json` 或 `pipeline-meta.json`
- 新资源进入游戏前，先检查是否与现有玻璃战术语言一致；宁可少，也不要风格混杂。

## 9. 当前实现备注

- `range_move`、`range_attack`、`range_danger`、`range_selected`、`selected_ring` 继续沿用玻璃战术资产。
- `MountedUiLayer` 仍可作为棋盘背景蒙版或底层挂板，但不再承担正式 HUD 主体。
- 正式 HUD 以场景节点为主，代码绘制只保留棋盘、单位与特效。
